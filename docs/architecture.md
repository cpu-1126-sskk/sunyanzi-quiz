# 技术架构与逻辑设计 (Architecture & Logic) - v2.1

本项目采用了一种非线性的、基于 **Likert 5 阶量表** 与 **指数级全维度共振** 的心理测量架构。

## 1. 核心逻辑流转 (Logical Flow)

### 阶段 I：全量题库抽样 (Sampling)
- **池化管理**: 32 道精心设计的 Likert 陈述句（包含反向测项）。
- **动态抽样**: 每次会话随机抽取 12 道题目，确保测试的复玩性与测量效度。

### 阶段 II：指数级全人格共振 (Scoring Model)
- **非线性映射**: 选项 1-5 映射为 `[0, 1.5, 3, 6, 10]` 权重。这种设计强化了“极端偏好”的视觉权重，使雷达图更具棱角。
- **影子权重 (Shadow Weights)**: 每一道题目对所有 6 个性格维度均有 0.1 的微小影响，确保星图永远闭合，彻底消除“硬零值”。
- **反向计分 (Reverse Coding)**: 系统自动识别负值权重，并执行 `[10, 6, 3, 1.5, 0]` 的翻转映射。

### 阶段 III：归一化与欧氏距离匹配 (Normalization & Matching)
- **动态分母**: `maxPossible` 根据当前会话实际抽取的题目实时计算，确保得分不因随机性而膨胀。
- **全局基准 (Global Floor)**: 引入 `GLOBAL_FLOOR = 20` 阈值，防止在样本不足时产生虚假高分。
- **命运扰动**: 在计算用户向量与 33 首歌曲原型的欧氏距离时，引入 `0.01` 的随机扰动，打破数学上的平局，增加“宿命感”。

## 2. 响应式视图渲染 (`src/main.js`)

- **3D 轨道系统**: 68度倾斜的开普勒轨道，引入 **LERP (Linear Interpolation)** 视差平滑算法，在 `requestAnimationFrame` 中实现丝滑动效并降低 CPU 负载。
- **空间美学**: 针对移动端长文本，采用 110px 最小高度容器，确保 4-5 行文本在小屏幕下仍能完美展示，避免布局塌陷。
- **资产清洗 (Asset Cleaner)**: 自动化映射机制，将复杂专辑名转化为标准化的 `.jpg` 文件名，增强系统容错性。

## 3. 媒体链路

- **iframe 鲁棒解析**: 正则表达式兼容 Markdown 引号包裹格式，确保从 B 站抓取的媒体资源能稳定嵌入。
- **100vh 垂直锁定**: 移动端采用全屏锁定布局，消除滚动条导致的视觉破碎感。
